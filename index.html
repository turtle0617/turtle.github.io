CSS Background - 擬亂數背景

Posted on 2018-07-09



CSS 沒有亂數及變數命名，那我們要如何達到用CSS呈現出亂數背景呢？
在此之前，先向大家介紹一個小知識

The Cicada Principle


The Cicada Principle - 蟬原則


“蟬原則”源自於北美的週期性蟬，在每7,13或17年出現，找到配偶並死亡。
那為什麼剛好是在7,13或17年出現呢？是還是刻意選擇好的還是純粹是剛好，這個問題便吸引了一部分科學家的好奇。
研究後發現這些數字確實是有共通處，這些數字都是所謂的質數，也就是自己才能剛好整除自己，但這跟質數又有什麼關係呢？
原來蟬的天敵鳥類、蜘蛛、黃蜂、魚和蛇，牠們的生命週期通常都是2~6年。
因此，如果蟬的生命週期是12年出現一次，那在2、3、4、6年都有機會遇到牠們的天敵。
還有一個很原因就是同區域的蟬並不止有一個種群，使用質數當作生命週期可以大大降低同時出現互相競爭捕食的局面。
所以這種以質數作為循環週期來增加”自然隨機性”的策略就稱之為”蟬原則”。

那這跟我們要做的亂數背景又有什麼關係呢？


我們都知道background 可以用 linear-gradient 產生條紋背景（如下圖）：



可以看到圖片的條紋背景有非常明顯的重複性。
那如果我們把每一個條紋都設一個背景區塊是否就能降低重複率呢？
廢話不多說，馬上來實作看看：

這變要注意到,linear-gradient在第一行的權限是最大的，可以覆蓋下面所有的顏色，
越往下寫權限越低，以此類推。

來玩玩看兩個例子吧

可以看到圖片中的重複率確實是比上一張圖低了許多，但我們還是可以看到在240px處後會開始產生重複。
為什麼是240px呢？這是因為當今天用多個背景區塊時重複率將會採所有區塊的最小公倍數。
可以看到我們剛剛用了80、60、40，最小公倍數為240，所以在240px後就會開始產生重複。
這時候我們就可以回到我們一開始的主題”蟬原則”，前面有提到過質數只能自己除以自己，所以今天用質數當作背景的大小時
最小公倍數就會是該數字的相成，讓我們來看看實例：

為了保持最大隨機數，在條紋的寬度也用質數去替代，避免條紋寬度最小公倍數過小。
範例

可以看到圖片中已經沒有任何地方有重複的了，
因為現在的最小公倍數為 41 x 61 x 83 = 207853，
也就是說要在207853px後才會看到第一個重複，但這也已經超過現有螢幕的最大解析度了。
